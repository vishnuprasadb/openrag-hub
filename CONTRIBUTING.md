## ğŸ“¦ Pull Requests

When submitting a pull request:

1. Ensure your branch is up-to-date with `main`
2. Follow the [branching strategy](#branching-strategy)
3. Include a clear description of the changes
4. Add or update documentation where relevant
5. Ensure the app starts without errors
6. Add tests if relevant
7. Link related issues if applicable

### PR Title Format

```
type: short description

body

footer
```

### Types

- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation
- `refactor`: Code change that neither fixes a bug or adds a feature
- `test`: Adding missing tests or correcting existing tests
